## What this page is for

This page defines the default working model at Helixiora: remote-first, low-bureaucracy, and explicit on outcomes.

## What you should do

- **Remote-first:** Write updates and decisions so teammates can follow async.
- **Flexible hours:** Keep shared availability visible in calendar and Slack status.
- **Async before sync:** Default to docs/messages, use meetings when speed or complexity requires it.
- **Focus time:** Protect blocks for deep work and signal when you are heads-down.
- **Ownership:** Drive work to completion and surface blockers early.
- **Definition of done:** Work is done when the outcome is delivered, documented, and handoff-ready.

## Who owns or approves it

Managers reinforce these norms in teams. People Operations and leadership approve company-wide working expectations.

## Where to go in the tool stack

- `Slack` for daily coordination and status visibility
- Project tracking tool for priorities and handoffs
- Internal docs for decision records and process notes

## What happens if something goes wrong

If expectations are unclear, ask your manager to clarify scope, urgency, and ownership. If blockers persist, escalate early rather than letting work stall silently.
